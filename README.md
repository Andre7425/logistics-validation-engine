# Logistics Validation Engine: NYC Taxi Multi-Database Analytics & ML Pipeline

Проєкт розроблено в межах дослідження методів обробки великих даних (Big Data), розгортання багатокомпонентних NoSQL інфраструктур та підготовки ознак (Feature Engineering) для моделей машинного навчання.

В основі рішення лежить автоматизація обробки датасету NYC Taxi з міграцією даних у різні типи СКБД (документні, графові, key-value) для оптимізації специфічних аналітичних запитів.

## 📁 Структура та архітектура модулів (`src/taxi_pipeline`)

Проєкт логічно розділений на кілька функціональних шарів:

### 1. Інфраструктурний шар (Data Ingestion & Migration)
* `to_mongodb.py` — Скрипт для міграції та структурування сирих даних у MongoDB (робота з часовими рядами, оптимізоване індексне сканування `IXSCAN`).
* `to_neo4j.py` — Трансформація даних у графову модель для аналізу топології поїздок, взаємозв'язків між районами та побудови графів маршрутів.
* `to_redis.py` — Налаштування шару кешування в Key-Value сховищі для миттєвого доступу до агрегованих метрик та гарячих даних.

### 2. Аналітичний шар (Feature Engineering & Analytics)
* `spatial_analytics.py` — Геопросторовий аналіз координат поїздок, робота з геозонами (Taxi Zones) та розрахунок дистанцій.
* `time_analytics.py` — Виділення часових ознак (годин пік, днів тижня, сезонності) за допомогою MongoDB Aggregation Pipelines.
* `stats.py` & `taxi_analytics.py` — Розрахунок ключових бізнес-метрик: середнього чека, розподілу чайових, щільності потоку таксі.
* `trip_jfk.py` — Спеціалізований логістичний аналіз поїздок до/з Міжнародного аеропорту імені Джона Кеннеді (JFK).

### 3. Шар Машинного Навчання (Machine Learning)
* `train_model.py` — Конвеєр передобробки даних та навчання предиктивної моделі для прогнозування вартості чи тривалості поїздки.
* `taxi_model.joblib` — Серіалізована (збережена) навчена ML-модель, готова до деплою в Production.

### 4. Інтерфейс та сервіси
* `app.py` — Головна точка входу (веб-інтерфейс або API-сервіс) для взаємодії з аналітичним пайплайном.
* `download_coords.py` & `taxi_zones_with_coords.csv` — Модуль завантаження та географічний довідник координат і меж таксомоторних зон Нью-Йорка.
* `test_pipeline.py` — Скрипт автоматизованого тестування працездатності всього ETL-конвеєра.

## 🛠️ Технологічний стек
* **Python 3.10+** (Pandas, PyMongo, Joblib, Scikit-Learn)
* **Docker / Docker Compose / WSL 2** (Ubuntu)
* **MongoDB** (Документо-орієнтована база, агрегаційні пайплайни)
* **Neo4j** (Графова база даних для логістичних маршрутів)
* **Redis** (In-memory кеш для швидкого відгуку системи)
